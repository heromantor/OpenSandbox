Define `VMConfig` struct with all exported fields:
- `ID              string`   -- VM identifier (UUID). If empty, auto-generated.
- `VCPUs           int64`    -- Number of virtual CPUs (1-32, default 1)
- `MemoryMiB       int64`    -- Memory in MiB (minimum 128, default 256)
- `KernelImagePath string`   -- Path to the guest kernel image (required)
- `RootfsPath      string`   -- Path to the root filesystem ext4 image (required)
- `KernelArgs      string`   -- Kernel boot arguments (default: "console=ttyS0 reboot=k panic=1 pci=off")

- `Jailer          JailerOpts` -- Jailer configuration (used when JailerEnabled=true)
- `LogLevel        string`   -- Firecracker log level: "Error", "Warning", "Info", "Debug" (default "Error")
- `FirecrackerBin  string`   -- Path to firecracker binary (default "/usr/bin/firecracker")
- `JailerBin       string`   -- Path to jailer binary (default "/usr/bin/jailer")

Add a `Validate() error` method on VMConfig that checks:

Add a `withDefaults() VMConfig` method that fills zero-value fields with defaults:

- MemoryMiB: 256
- KernelArgs: "console=ttyS0 reboot=k panic=1 pci=off"

1. Create `runtime/firecracker/jailer.go` (package `firecracker`):

Define `JailerOpts` struct:
- `UID           int`      -- Unprivileged user ID for jailed process (required, must be > 0)
- `GID           int`      -- Unprivileged group ID for jailed process (required, must be > 0)
- `ChrootBaseDir string`   -- Base chroot directory (default "/srv/jailer")
- `CgroupVersion string`   -- "1" or "2" (auto-detected if empty)
- `Daemonize     bool`     -- Run as daemon (default true)
- `NumaNode      int`      -- NUMA node to pin to (-1 = no pinning, default -1)

Define `detectCgroupVersion() string` function:
- Check if `/sys/fs/cgroup/cgroup.controllers` exists. If yes, return "2".
- Otherwise return "1".
- Use `os.Stat()`, not `os.ReadFile()`.

Define `chrootDir(baseDir, vmID string) string` function (not exported):
- Returns `filepath.Join(baseDir, "firecracker", vmID, "root")`
- This matches the Jailer convention: `<base>/<exec-name>/<id>/root/`

Define `socketPathInChroot() string` function (not exported):
- Returns `"run/firecracker.socket"` -- relative to chroot root, as per Jailer convention.

Define `validateJailerOpts(opts JailerOpts) error` function (not exported):
- UID must be > 0
- GID must be > 0
- ChrootBaseDir must not be empty
- Full socket path (`/srv/jailer/firecracker/<uuid>/root/run/firecracker.socket`) must be <= 108 chars. Compute expected length using a 36-char UUID placeholder and validate.

Define `buildJailerConfig(vmID string, opts JailerOpts, firecrackerBin string) (JailerConfig, error)` -- this is an exported function that returns the firecracker-go-sdk `JailerConfig` type.
Wait -- to avoid importing the SDK types in the public API, instead return a struct that can be converted later. But the research says to use the SDK directly. Since Plan 02 will do the actual SDK call, for now define a `resolvedJailerConfig` unexported struct that mirrors the SDK fields:
- `id            string`
- `uid           int`
- `gid           int`
- `execFile      string`
- `chrootBaseDir string`
- `cgroupVersion string`
- `daemonize     bool`
- `numaNode      int`

Define `resolveJailerConfig(vmID string, opts JailerOpts, firecrackerBin string) (*resolvedJailerConfig, error)`:
- Fill defaults (ChrootBaseDir -> "/srv/jailer", CgroupVersion -> detectCgroupVersion(), Daemonize -> true, NumaNode -> -1 if 0)
- Call validateJailerOpts
- Return resolved config

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VMConfig input | User-supplied configuration values (vCPUs, memory, paths) before VM creation |
| Filesystem paths | Kernel image path, rootfs path, custom CPU template path -- could reference arbitrary host files |
| Jailer UID/GID | Privilege-related values that affect the security posture of the jailed Firecracker process |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | VMConfig.KernelImagePath | mitigate | ValidateKernelImage checks file exists and is non-empty; Plan 02 will add checksum verification against KernelManifest.Checksum |
| T-01-02 | Elevation of Privilege | JailerOpts.UID/GID | mitigate | validateJailerOpts enforces UID > 0 and GID > 0, preventing root (UID 0) execution inside jailer |
| T-01-03 | Denial of Service | VMConfig.VCPUs/MemoryMiB | mitigate | VMConfig.Validate() enforces VCPUs in [1,32] and MemoryMiB >= 128, preventing unbounded resource allocation |
| T-01-04 | Information Disclosure | Socket path length | mitigate | validateJailerOpts checks combined path <= 108 chars to prevent socket creation failure that could leave partial state |
| T-01-05 | Denial of Service | Cleanup failure | accept | VMResources.Cleanup uses multierror to attempt all cleanups; partial cleanup is logged. Orphan GC deferred to Plan 02 startup scan. |
</threat_model>
