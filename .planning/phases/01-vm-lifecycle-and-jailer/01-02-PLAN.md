1. **Add `toFirecrackerConfig` method to `vm.go`:**

Add import for the firecracker-go-sdk. Use an alias to avoid collision with the package name:
```go
import (
    sdk "github.com/firecracker-microvm/firecracker-go-sdk"
    "github.com/firecracker-microvm/firecracker-go-sdk/client/models"
)
```

Add `socketPath() string` method on VMConfig:
- If JailerEnabled: return `socketPathInChroot()` (relative, jailer resolves the rest)
- If not JailerEnabled: return `filepath.Join(os.TempDir(), fmt.Sprintf("firecracker-%s.socket", c.ID))`

Add `toFirecrackerConfig() sdk.Config` method on VMConfig:
- Build drives: `sdk.NewDrivesBuilder(c.RootfsPath).Build()`
- Set `VMID: c.ID`
- Set `SocketPath: c.socketPath()`
- Set `KernelImagePath: c.KernelImagePath`
- Set `KernelArgs: c.KernelArgs`
- Set `MachineCfg` with `VcpuCount: sdk.Int64(c.VCPUs)`, `MemSizeMib: sdk.Int64(c.MemoryMiB)`, `TrackDirtyPages: sdk.Bool(c.TrackDirtyPages)`
- If `c.CPUTemplate.Static != TemplateNone`: set `MachineCfg.CPUTemplate = models.CPUTemplate(c.CPUTemplate.Static)`
- If `c.JailerEnabled`: build JailerCfg by calling `resolveJailerConfig(c.ID, c.Jailer, c.FirecrackerBin)` and translating to `sdk.JailerConfig`:
  ```go
  uid := resolved.uid
  gid := resolved.gid
  cfg.JailerCfg = &sdk.JailerConfig{
      ID:            resolved.id,
      UID:           &uid,
      GID:           &gid,
      ExecFile:      resolved.execFile,
      ChrootBaseDir: resolved.chrootBaseDir,
      CgroupVersion: resolved.cgroupVersion,
      Daemonize:     resolved.daemonize,
  }
  ```
- If `resolved.numaNode >= 0`: set `cfg.JailerCfg.NumaNode = &resolved.numaNode`
- Return the config

**Implement `Create(ctx, cfg VMConfig) (*VM, error)`:**
- Call `cfg.withDefaults()` to fill missing fields
- Call `cfg.Validate()` -- return wrapped error on failure
- Call `cfg.CPUTemplate.Validate()` -- return wrapped error on failure
- Call `ValidateKernelImage(cfg.KernelImagePath)` -- return error if kernel missing
- Check `m.vms[cfg.ID]` -- return `VMAlreadyExistsError` if exists
- Build `fcCfg := cfg.toFirecrackerConfig()`
- Create a `logrus.NewEntry(logrus.New())` logger. Set level based on cfg.LogLevel.
- Call `sdk.NewMachine(ctx, fcCfg, sdk.WithLogger(logger))` -- return `VMStartError` on failure
- Build `VMResources`:
  - `SocketPath`: full socket path (if jailer: `chrootDir(opts.ChrootBaseDir, cfg.ID) + "/run/firecracker.socket"`, else: `cfg.socketPath()`)
  - `ChrootDir`: if JailerEnabled, `chrootDir(opts.ChrootBaseDir, cfg.ID)`, else empty
- Create `VM` struct with State=StateCreated, Config=cfg, SocketPath, Resources
- Store in `m.vms[cfg.ID]` and `m.machines[cfg.ID]`
- Return &VM

**Implement `Start(ctx, vmID string) error`:**
- Lock mutex, get VM from `m.vms[vmID]` -- return `VMNotFoundError` if missing
- Check State == StateCreated -- return error if not
- Set State = StateStarting
- Get machine from `m.machines[vmID]`
- Call `machine.Start(ctx)` -- on error, set State back to StateCreated, return `VMStartError`
- Set `vm.PID = machine.PID()`
- Set State = StateRunning
- Return nil

**Implement `Stop(ctx, vmID string) error`:**
- Lock mutex, get VM -- return `VMNotFoundError` if missing
- Check State == StateRunning -- return error if not Running
- Set State = StateStopping
- Get machine from `m.machines[vmID]`
- Try `machine.Shutdown(ctx)` first (graceful Ctrl+Alt+Del)
- If shutdown fails, try `machine.StopVMM()` (SIGTERM)
- Call `machine.Wait(ctx)` to wait for exit (ignore "process already exited" errors)
- Set State = StateStopped
- Set PID = 0
- Return nil (do NOT clean up resources here -- that's Destroy's job)

**Implement `Destroy(ctx, vmID string) error`:**
- Lock mutex, get VM -- return `VMNotFoundError` if missing
- If State == StateRunning, call Stop first (unlock, call Stop, re-lock)
- Call `vm.Resources.Cleanup()` -- collect error but don't fail on cleanup errors
- Delete from `m.vms[vmID]` and `m.machines[vmID]`
- Return cleanup error if any (as `CleanupError`)

**Implement `Get(ctx, vmID string) (*VM, error)`:**
- RLock mutex, get VM -- return `VMNotFoundError` if missing
- Return copy of VM (not the pointer to internal state). Create a shallow copy:
  ```go
  vmCopy := *vm
  return &vmCopy, nil
  ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VMConfig -> SDK Config | User-supplied values translated to Firecracker SDK configuration |
| Manager -> Firecracker process | SDK starts a child process; must ensure it runs jailed |
| Stop/Destroy -> filesystem | Cleanup removes files/directories; must not delete outside chroot |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | Manager.Create without Jailer | mitigate | JailerEnabled defaults are documented; production deployment guide will mandate JailerEnabled=true. Code logs warning if JailerEnabled=false. |
| T-02-02 | Tampering | VMConfig.RootfsPath pointing outside expected dir | accept | Path validation deferred to Phase 2 (rootfs management). Phase 1 accepts any valid file path. Risk: low -- runtime is server-side, not user-facing directly. |
| T-02-03 | Denial of Service | Stop() hangs if Firecracker process ignores SIGTERM | mitigate | Stop uses context with deadline; Shutdown (Ctrl+Alt+Del) first, then StopVMM (SIGTERM), then context timeout cancels Wait. |
| T-02-04 | Denial of Service | Leaked resources on crash (stale chroot) | accept | Orphan chroot cleanup deferred to startup GC scan (addressed in research as Pitfall 3). Phase 1 generates unique UUIDs so re-creation never collides. |
| T-02-05 | Information Disclosure | VM state readable via Get() | accept | Manager is server-internal; no network-facing API in Phase 1. Access control handled at the API layer in later phases. |
</threat_model>
