   - `DefaultSubnetBase` = `0xAC100000` (172.16.0.0)
   - `MaxSubnetIndex` = `16383` (16384 /30 subnets in 172.16.0.0/16)
   - `DefaultNameservers` = `[]string{"8.8.8.8", "8.8.4.4"}`
   - `maxTAPNameLen` = 15 (IFNAMSIZ - 1)

2. **SubnetAllocation struct:**
   ```go
   type SubnetAllocation struct {
       HostIP    net.IP
       GuestIP   net.IP
       Subnet    net.IPNet
       GatewayIP net.IP
   }
   ```

3. **AllocateSubnet(index uint32) (SubnetAllocation, error):**
   - If index > MaxSubnetIndex, return error
   - offset = index * 4
   - hostIP = base + offset + 1
   - guestIP = base + offset + 2
   - subnet = base + offset with /30 mask
   - gatewayIP = hostIP
   - Convert uint32 to net.IP using encoding/binary.BigEndian

4. **GenerateMAC() (string, error):**
   - Use `crypto/rand.Read(buf[:4])` for 4 random bytes
   - Prefix with `02:FC:` (locally-administered, Firecracker mnemonic)
   - Format as `02:FC:%02x:%02x:%02x:%02x`

5. **TAPDeviceName(vmID string) string:**
   - Prefix "tap-"
   - Truncate vmID to first 11 chars (4 + 11 = 15 = IFNAMSIZ - 1)
   - If vmID shorter than 11, use full vmID

6. **NetworkConfig struct:**
   ```go
   type NetworkConfig struct {
       SubnetIndex   uint32
       HostInterface string   // host outbound interface for iptables rules
       Nameservers   []string // DNS nameservers for guest (max 2)
   }
   ```

7. **NetworkConfig.Validate() error:**
   - If HostInterface is empty, return InvalidVMConfigError{Field: "NetworkConfig.HostInterface"}
   - If SubnetIndex > MaxSubnetIndex, return error
   - If len(Nameservers) > 2, return error (SDK limit)

8. **NATRules struct:**

   type NATRules struct {
       GuestIP   string

   }

Create `runtime/firecracker/network_linux.go` with `//go:build linux`:

1. **CreateTAPDevice(name string, hostIP net.IP, subnet net.IPNet) error:**
   - Create TAP: `netlink.Tuntap{LinkAttrs: netlink.LinkAttrs{Name: name}, Mode: netlink.TUNTAP_MODE_TAP}`
   - `netlink.LinkAdd(tap)` -- wrap errors with `fmt.Errorf("firecracker: create tap %s: %w", name, err)`
   - Find link by name: `netlink.LinkByName(name)`
   - Assign address: `netlink.AddrAdd(link, &netlink.Addr{IPNet: &net.IPNet{IP: hostIP, Mask: subnet.Mask}})`
   - Bring up: `netlink.LinkSetUp(link)`
   - On any error after LinkAdd, attempt cleanup: `netlink.LinkDel` the partially-created device

2. **DeleteTAPDevice(name string) error:**
   - `netlink.LinkByName(name)` -- if error (device gone), return nil (idempotent)
   - `netlink.LinkDel(link)` -- wrap errors

3. **NATRules.Apply() error:** (method on NATRules from network.go)
   - Create `iptables.New()` -- handles iptables-nft auto-detection
   - `ipt.AppendUnique("nat", "POSTROUTING", "-o", r.HostIface, "-s", r.GuestIP, "-j", "MASQUERADE")`
   - `ipt.AppendUnique("filter", "FORWARD", "-i", r.TAPName, "-o", r.HostIface, "-j", "ACCEPT")`
   - Wrap each error with `fmt.Errorf("firecracker: add nat rule: %w", err)`

4. **NATRules.Remove() error:**
   - Create `iptables.New()`
   - `ipt.Delete("nat", "POSTROUTING", ...)` -- ignore "rule not found" errors (idempotent)
   - `ipt.Delete("filter", "FORWARD", ...)` -- ignore "rule not found" errors
   - Return nil unless unexpected error

5. **EnsureIPForwarding() error:**
   - Read `/proc/sys/net/ipv4/ip_forward`
   - If content is "0\n" or "0", return descriptive error: `"firecracker: ip forwarding disabled: set net.ipv4.ip_forward=1 via sysctl"`
   - If "1", return nil

6. **EnsureSharedForwardRule(hostIface string) error:**
   - `ipt.AppendUnique("filter", "FORWARD", "-m", "conntrack", "--ctstate", "RELATED,ESTABLISHED", "-j", "ACCEPT")`
   - Called once at Manager startup, not per-VM

7. **DefaultHostInterface() (string, error):** (Linux implementation)
   - `netlink.RouteList(nil, netlink.FAMILY_V4)` 
   - Find route where `Dst == nil` (default route)
   - Get link by index: `netlink.LinkByIndex(route.LinkIndex)`
   - Return link.Attrs().Name
   - If no default route found, return error

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| guest VM -> host TAP | Guest network traffic crosses from VM to host network stack via TAP device |
| host TAP -> internet | NAT'd traffic from guest exits host via iptables MASQUERADE |
| guest DNS -> nameserver | Guest DNS queries go to configured nameservers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | Guest IP source spoofing | mitigate | /30 subnet limits guest to single IP; iptables MASQUERADE scoped to specific source IP in NATRules.Apply() |
| T-04-02 | Tampering | Guest ARP poisoning other VMs | mitigate | /30 point-to-point subnet (only host+guest on L2 segment); no shared bridge between VMs |
| T-04-03 | Information Disclosure | Stale iptables rules after crash | mitigate | Idempotent NATRules.Remove() in cleanup; orphan detection planned for Manager startup |
| T-04-04 | Denial of Service | TAP device name collision | mitigate | TAP name derived from VM UUID (guaranteed unique by Manager); CreateTAPDevice fails if name exists |
| T-04-05 | Elevation of Privilege | Guest bypasses egress proxy via raw IP | accept | DNS-based enforcement only in current design; IP-level iptables DROP is a future enhancement; documented as known limitation |
| T-04-06 | Input Validation | NetworkConfig field validation | mitigate | NetworkConfig.Validate() checks HostInterface non-empty, SubnetIndex in range, Nameservers count <= 2; called before TAP creation |
</threat_model>
