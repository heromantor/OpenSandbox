```go
func (r *VMResources) cleanupNetwork() error {
    var result *multierror.Error
    if r.TAPDeviceName != "" {
        if err := DeleteTAPDevice(r.TAPDeviceName); err != nil {
            result = multierror.Append(result,
                fmt.Errorf("remove tap %s: %w", r.TAPDeviceName, err))
        }
    }
    if r.NATRules != nil {
        if err := r.NATRules.Remove(); err != nil {
            result = multierror.Append(result,
                fmt.Errorf("remove nat rules: %w", err))
        }
    }
    return result.ErrorOrNil()
}
```

In Manager.Create, after CID auto-assignment and before Validate:

```go
// If NetworkConfig is provided, resolve host interface and subnet.
if cfg.NetworkConfig != nil {
    // Auto-assign subnet index if not explicitly set.
    // (SubnetIndex 0 is valid, so we need a separate auto-assign mechanism)
    // Actually, SubnetIndex should always be auto-assigned by the manager.
    cfg.NetworkConfig.SubnetIndex = m.subnetAlloc.Allocate()

    // Resolve host interface if not configured.
    if cfg.NetworkConfig.HostInterface == "" {
        if m.config.HostInterface != "" {
            cfg.NetworkConfig.HostInterface = m.config.HostInterface
        } else {
            iface, err := DefaultHostInterface()
            if err != nil {
                return nil, fmt.Errorf("firecracker: create vm: detect host interface: %w", err)
            }
            cfg.NetworkConfig.HostInterface = iface
        }
    }

    // Prepend egress proxy address as first nameserver if configured.
    if m.config.EgressProxyAddr != "" {
        cfg.NetworkConfig.Nameservers = append(
            []string{m.config.EgressProxyAddr},
            cfg.NetworkConfig.Nameservers...,
        )
        // Trim to max 2 nameservers (SDK/kernel limit).
        if len(cfg.NetworkConfig.Nameservers) > 2 {
            cfg.NetworkConfig.Nameservers = cfg.NetworkConfig.Nameservers[:2]
        }
    }
}
```

}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Manager.Create -> host network stack | TAP creation and iptables rules modify host networking |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Spoofing | Egress proxy DNS address injection | mitigate | EgressProxyAddr configured in ManagerConfig (operator-controlled); validated as part of NetworkConfig nameserver list |
| T-04-08 | Tampering | Manager.Create network rollback | mitigate | If sdk.NewMachine fails, TAP device and NAT rules are cleaned up in error path to prevent resource leaks |
| T-04-09 | Denial of Service | Subnet index exhaustion | accept | SubnetIndexAllocator is monotonic; 16384 concurrent VMs is the hard limit from 172.16.0.0/16; documented in MaxSubnetIndex constant |
| T-04-10 | Information Disclosure | VMResources.NATRules contains guest IP | accept | NATRules struct only contains network-scoped IPs (172.16.x.x); no PII or secrets |
| T-04-11 | Elevation of Privilege | cleanupNetwork on non-Linux | mitigate | cleanup_other.go returns nil on non-Linux; prevents accidental Linux-only syscalls on wrong platform |
</threat_model>
