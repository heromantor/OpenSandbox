<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host -> Guest vsock | Untrusted guest code runs behind the vsock device; CID must be unique to prevent cross-VM access |
| UDS filesystem path | Socket files on host filesystem; stale files could be hijacked if not cleaned up |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | CIDAllocator | mitigate | Atomic counter starting at MinGuestCID=3 guarantees uniqueness; Firecracker hypervisor enforces CID isolation at device level |
| T-03-02 | Tampering | vsock UDS path | mitigate | Paths constructed programmatically from validated VM IDs (regex `^[a-zA-Z0-9-]+$` prevents path traversal); stale sockets removed by VMResources.Cleanup() |
| T-03-03 | Denial of Service | UDS path length | mitigate | Validate path length <= 108 in VMConfig.Validate() to prevent socket creation failure |
| T-03-04 | Information Disclosure | CID value | accept | CID is a monotonic counter visible to the guest; no sensitive information leaked since CID is just a routing identifier |
</threat_model>
