creation path.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VMConfig input -> SDK Config | User-provided VsockCID must be validated before passing to SDK |
| Manager -> CIDAllocator | CID allocation must be atomic to prevent races in concurrent Create calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Spoofing | toFirecrackerConfig VsockDevices | mitigate | CID validated >= MinGuestCID in VMConfig.Validate() (Plan 01); SDK validates minimum:3 at API level |
| T-03-06 | Denial of Service | Manager.Create CID exhaustion | accept | Atomic Uint32 counter wraps at 2^32; no realistic process creates 4 billion VMs; restart resets counter and old CIDs are freed |
| T-03-07 | Tampering | VsockUDSPath in VMResources | mitigate | Path constructed programmatically from validated VM ID; cleanup removes socket file on Destroy |
</threat_model>
